# Whishper YouTube Captions

Use your local Whishper server as a real YouTube subtitle provider.

This browser extension adds a native `Whishper` entry to YouTube's original subtitles menu. Pick it like any other YouTube caption track, and subtitles generated by your local Whishper backend are rendered by YouTube's own caption system.

## Why Use It

- Native YouTube experience: `Whishper` appears inside YouTube's built-in subtitles menu, not as a separate floating overlay.
- Local-first transcription: sends the current YouTube URL to your own Whishper backend instead of a hosted subtitle service.
- Works when YouTube has no captions: videos without original subtitles still get a `Whishper` subtitle option.
- Optional translated subtitles: configure a target language and play Whishper's translated segments directly in YouTube.
- Reuses existing jobs: previously created Whishper jobs are restored automatically for the same video.
- No custom subtitle window: rendering goes through YouTube's captions pipeline, so positioning, styling, and fullscreen behavior remain familiar.

## How It Works

1. Open a YouTube video.
2. The extension adds `Whishper` to YouTube's native subtitle list.
3. Select `Whishper` from YouTube Settings -> Subtitles.
4. The extension asks your local Whishper backend to transcribe the video URL.
5. When the job is ready, the extension converts Whishper segments into YouTube-compatible timed text.
6. YouTube renders the captions using its normal caption renderer.

If a target translation language is configured, the extension requests Whishper's translation for that job and plays the translated subtitle segments instead of the original transcription.

## Requirements

- A Chromium-based browser or Firefox-compatible WXT build.
- A running Whishper backend reachable from the browser.
- YouTube watch pages, for example `https://www.youtube.com/watch?v=...`.

Default backend URL:

```text
http://localhost:8082
```

## Install

Build the extension:

```sh
npm install
npm run build
```

Load it in Chrome or another Chromium browser:

1. Open `chrome://extensions`.
2. Enable Developer mode.
3. Click Load unpacked.
4. Select `extension/.output/chrome-mv3`.

For local development:

```sh
npm run dev
```

Then load the generated development output from `.output`.

## Configure

Open the extension options page and set:

- Backend API host: your Whishper backend, default `http://localhost:8082`.
- Language: source transcription language, default `auto`.
- Subtitle target language: optional translation target, blank for original transcription.
- Model size: Whisper model size, default `base`.
- Device: `cpu` or `cuda`.
- Poll interval: how often the extension checks job status, default `5000` ms.

Translation examples:

```text
zh
fr
ja
es
```

Leave Subtitle target language blank if you want the original transcribed language.

## Use

1. Start your Whishper backend.
2. Open a YouTube video.
3. Open YouTube's Settings menu.
4. Choose Subtitles.
5. Select Whishper.
6. Wait for transcription or translation to finish.

The extension status panel shows whether Whishper is queued, transcribing, translating, ready, or failed.

## What You Should See

On videos with existing captions:

```text
Subtitles (N)
  Off
  Whishper
  English
  ...
```

On videos without captions:

```text
Subtitles (1)
  Off
  Whishper
```

After selecting `Whishper`, captions should appear in YouTube's normal caption area. There should be no separate extension subtitle overlay.

## Privacy

- The extension sends the YouTube page URL to the configured Whishper backend.
- Transcription and translation happen through your Whishper deployment.
- No hosted extension service is required by this extension.
- Storage is limited to extension settings and cached job IDs per YouTube video.

## Troubleshooting

Whishper does not appear in YouTube's subtitle menu:

- Reload the extension in `chrome://extensions`.
- Open a fresh YouTube tab after reloading.
- Confirm the extension is enabled for YouTube.
- Wait for the player to finish loading, especially if an ad is playing.

Transcription never starts:

- Select `Whishper` from the YouTube subtitle menu.
- Confirm the backend URL is correct in extension options.
- Check that `http://localhost:8082/api/transcriptions` is reachable from the browser.

Translation does not appear:

- Confirm the target language is supported by your Whishper translation service.
- Try leaving Subtitle target language blank to verify original transcription works.
- Check that Whishper's translation service is running.

Chrome ignores command-line extension loading:

- Some Google Chrome builds ignore `--load-extension`.
- Load or reload `extension/.output/chrome-mv3` manually from `chrome://extensions`.

## For Developers

This extension is built with WXT, TypeScript, and OXC tooling via `oxlint`.

Useful commands:

```sh
npm run check
npm run lint
npm run build
```

The extension uses Whishper's existing backend endpoints:

- `POST /api/transcriptions`
- `GET /api/transcriptions/:id`
- `GET /api/translate/:id/:target`

When creating transcription jobs, the extension uses the backend form field `sourceUrl`.
